Process Description

The diagram above illustrates the process of creating a GitHub repository from a local folder in the Prasso application:

Client-Side (Browser)

  • Folder Selection: User clicks the "Browse for Folder" button and selects a folder from their local file system.
  • File Processing: JavaScript reads the folder contents and extracts the folder name to suggest as the repository name.
  • Upload Preparation: When the user confirms, the application creates a FormData object containing all the files with their relative paths preserved.
  • File Upload: Files are uploaded to the server with progress indicators.

Server-Side

  • File Reception: The server receives the uploaded files and validates the request.
  • Temporary Storage: A unique temporary directory is created to store the uploaded files.
  • Structure Preservation: The folder structure from the client is preserved in the temporary directory.
  • GitHub API Interaction: The server creates a new GitHub repository using the GitHub API.
  • Git Operations: The server initializes a Git repository in the temporary directory, adds the remote origin, commits all files, and pushes to GitHub.
  • Database Update: The site record is updated with the new GitHub repository path.

Response Handling

  • Success Response: The server returns a success response with details about the created repository.
  • User Notification: The user is shown a success message with repository details.
  • Page Refresh: The page is refreshed to show the updated site information.
